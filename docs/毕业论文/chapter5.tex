\chapter{实验设计与结果分析}

前四章已经分别从研究背景、相关技术、系统需求和平台实现等角度完成了全文主体框架的构建。对于一篇系统设计与实现型毕业论文而言，若缺少实验验证环节，则前述设计与实现难以形成完整闭环。因此，本章围绕“系统是否真正可用、各类能力是否能够协同工作、当前方案的优势与不足分别体现在哪里”三个问题展开实验设计与结果分析。

本文的实验部分根据当前系统成熟度进行取舍。考虑到本地视频检测模型尚未完全补齐、样本规模仍在扩充，同时整篇论文需要控制在合理篇幅内，本文不追求对所有任务给出大规模、全覆盖的定量结果，而是采用“图像虚假检测以定量评估框架和验证样本为主，视频与不安全内容检测以功能验证和案例分析为主，系统性能以可用性分析为主”的实验策略。这样既能保证实验内容与当前平台完成度保持一致，也能突出本文最具代表性的系统能力。

\section{实验目标与取舍原则}

结合本文平台的实现情况，本章实验主要服务于以下三个目的。第一，验证图像与视频检测链路能够真实跑通，平台不仅具备页面交互能力，也能够完成从输入到结果展示的完整闭环。第二，分析本地模型与外部接口在不同任务中的适用差异，为后续方法扩展和实验优化提供依据。第三，评估当前平台在研究验证和答辩演示场景下的可用性，包括任务响应特征、风险展示效果和系统组织方式等。

在实验取舍方面，本文优先保留三类最有代表性的验证内容。其一，是图像虚假内容检测实验。该任务既有本地模型，也有外部多模态接口，且仓库中已经包含可用于验证的图像测试样本，因此最适合作为平台检测能力的核心观察对象。其二，是视频虚假检测与不安全内容审核实验。虽然这一部分当前难以形成与图像任务同等规模的标准数据集对比，但其功能链路完整，能够体现平台对多模态任务的支持能力。其三，是系统性能与可用性分析。本文课题本质上是一个多源能力集成平台，因此系统在任务组织、前后端配合和同步/异步流程处理方面的表现，同样构成实验评价的重要部分。

与之对应，本文暂不将大规模跨数据集、跨模型的综合排行榜式对比作为实验重点，原因主要有两点。一方面，当前平台阶段更强调闭环实现与方法接入，尚未完成所有本地检测模型的扩展复现；另一方面，在本科毕业论文范围内，与其堆砌大量来源不完全一致的对比数值，不如将现有实验做得更真实、更可复验、更能反映系统特征。因此，本章的写作重点是“基于当前阶段完成度的可信实验分析”，而不是“追求形式上的指标齐全”。

\section{实验环境与评价指标}

\subsection{实验环境}

本文实验主要在校内实验室服务器环境下完成，平台采用前后端分离与模型服务独立运行的部署方式。前端页面通过静态构建方式部署并提供统一访问入口，后端代理服务负责接收前端请求并转发至本地模型服务或外部接口，本地模型服务则分别承担图像虚假检测、本地文生图和本地文生视频等推理任务。根据当前平台部署结构，前端静态页面运行于独立端口，后端代理服务和本地模型服务分别运行于不同端口并由独立进程管理。

从软件栈看，前端基于React、TypeScript、Vite和Ant Design构建；后端基于Node.js与Express实现；本地推理服务基于Python、FastAPI与PyTorch封装。外部能力主要来自图像与视频理解接口、图像与视频生成接口以及视觉智能类服务。这样的实验环境既反映了本文原型平台的真实部署方式，也为不同能力的统一接入提供了基础。

需要指出的是，本文实验的重点不在于评估底层硬件极限性能，而在于验证当前平台方案在研究和演示场景中的可行性。因此，本章更关注功能链路是否完整、结果表示是否稳定以及不同能力在统一平台中的适配情况，而不是对底层GPU吞吐能力进行专门测试。

\subsection{评价指标}

对于图像虚假内容检测任务，本文采用分类任务中常见的准确率、精确率、召回率和F1值作为主要评价指标。其定义如下：

\begin{equation}
Accuracy = \frac{TP + TN}{TP + TN + FP + FN}
\end{equation}

\begin{equation}
Precision = \frac{TP}{TP + FP}
\end{equation}

\begin{equation}
Recall = \frac{TP}{TP + FN}
\end{equation}

\begin{equation}
F1 = \frac{2 \times Precision \times Recall}{Precision + Recall}
\end{equation}

其中，$TP$ 表示将伪造样本正确识别为伪造的数量，$TN$ 表示将真实样本正确识别为真实的数量，$FP$ 表示将真实样本误判为伪造的数量，$FN$ 表示将伪造样本误判为真实的数量。上述指标适合用于衡量图像检测模型在二分类任务中的整体表现，尤其适合后续对不同检测方法进行对比。

对于视频检测与不安全内容审核任务，考虑到当前阶段更强调平台可用性验证和案例分析，因此本章不强行引入不完全匹配的数值指标，而主要从以下角度进行评价：第一，任务链路是否能够完整跑通；第二，返回结果是否具有结构化和可解释性；第三，平台能否将检测或审核结论稳定映射为统一的风险展示形式。这样的评价方式更符合当前平台进展，也能避免在数据基础不足的情况下形成不可靠数值。

\section{数据集与测试样本构建}

\subsection{图像真假检测样本}

结合当前仓库内容与平台进展，本文图像真假检测实验主要采用两类样本来源。第一类是仓库中已经整理完成的ProGAN测试样本集，当前共包含真实图像200张、伪造图像200张，总计400张样本。该数据集的优点在于标签明确、结构清晰，适合用于验证图像虚假检测接口与本地模型服务的基本可用性。第二类是平台生成与展示过程中使用的样例图像，包括平台内部文生图结果、人脸编辑结果以及若干代表性示例图片。这类样本虽然不用于严格定量统计，但有助于观察系统在开放场景输入下的判定表现。

从实验设计角度看，ProGAN测试样本适合作为当前阶段的“基线验证集”，其目的不是覆盖所有最新扩散模型，而是确认平台中的图像虚假检测链路能够在明确标签数据上稳定工作。与此同时，开放场景样例图像则用于弥补标准集与真实业务场景之间的差距，使实验分析能够同时覆盖结构化测试与案例观察两个维度。

\subsection{视频与安全审核样本}

与图像任务相比，视频与不安全内容审核实验当前更适合采用“代表性样例 + 功能链路验证”的方式进行。仓库中已经包含若干图像与视频样例，例如普通自然场景图像、明显风险场景图像、短视频样例以及生成或伪造相关样例素材。这些内容能够覆盖安全图像、风险图像、普通视频、疑似生成视频等几种典型输入类型，为验证平台在不同页面和不同任务下的行为提供了基础。

此外，平台自身具备内容生成能力，因此实验样本还可以来自平台内部生成结果。这样一来，第五章实验实际上继承了本文平台“生成—检测—展示”的闭环特点：部分样本来源于公开数据和仓库素材，部分样本来源于平台内部生成结果，两者共同构成当前阶段的实验验证基础。

\subsection{实验样本组织策略}

为了使实验部分保持紧凑且具有代表性，本文对样本组织采取了分层策略。对于能够形成明确标签和定量指标的任务，如图像虚假内容检测，优先使用结构化样本集进行评估；对于当前难以形成统一标准集、但平台链路已经完整的任务，如视频真假检测与内容安全审核，则采用代表性案例验证方式。这样的组织策略既体现了实验设计的针对性，也符合本科毕业论文在有限篇幅内突出核心成果的写作要求。

\section{图像虚假内容检测实验与分析}

\subsection{实验目的与方案}

图像虚假内容检测实验主要验证两个问题。第一，本地检测模型接入后，平台是否能够在结构化图像样本上完成稳定判断；第二，外部多模态图像AI鉴别接口在开放场景中能否补充本地模型在解释性和泛化性方面的不足。围绕这两个问题，本文将图像实验划分为“本地模型基线验证”和“开放场景样例分析”两部分。

在本地模型基线验证部分，实验使用ProGAN测试样本集，重点考察本地图像检测接口在统一数据集上的可用性以及准确率、精确率、召回率和F1值等指标的适用性。由于当前阶段论文重点仍然是系统平台实现与实验框架构建，本文不将这一部分展开为多轮大规模重复实验，而是将其作为后续补充定量对比的标准入口。在开放场景样例分析部分，则使用平台生成图片、示例图片和典型场景图片，观察本地检测模型与外部多模态接口在结果表达与适用范围上的差异。

\subsection{阶段性结果分析}

从当前平台验证情况看，本地图像虚假检测模块在结构化样本输入场景中具有较好的研究基线价值。一方面，本地模型服务接口清晰，适合接入统一评估流程；另一方面，后端代理已经能够将输入图像统一转发至本地服务，并将结果返回前端展示，这使得平台具备了开展后续批量评估与实验对比的条件。换言之，即使当前阶段尚未在论文中展开完整大表格统计，图像检测的实验框架已经具备了向严格量化实验过渡的基础。

与此相比，外部多模态图像AI鉴别接口更适合承担开放场景分析任务。其优势不在于替代本地基线模型，而在于能够提供相对灵活的自然语言解释和对复杂场景的适配能力。从平台使用体验上看，外部接口与本地模型在图像任务中的关系更接近互补而非竞争：前者更适合开放域案例分析和多模态推理，后者更适合作为稳定、可控的研究基线。

综合分析可知，图像虚假检测实验已经体现出本文平台方案的一个关键优势，即它并不依赖单一检测路径，而是通过统一代理层将“可控的本地方法”和“灵活的外部能力”组织到同一平台中。这一特征在后续补充更多图像检测模型后，将进一步增强平台的研究深度。

\section{视频检测与安全审核验证}

\subsection{视频真假检测验证}

视频真假检测实验的重点不在于形成大规模数字对比，而在于验证平台是否能够完成视频输入、统一转发、结构化结果返回和前端展示的完整链路。结合当前平台进展，视频真假检测主要通过多模态视频理解接口实现。实验中，用户既可以输入视频链接，也可以上传本地短视频，由后端统一转换为适配接口的形式，再返回真假结论、置信度与文本说明。

从阶段性结果来看，平台已经能够较稳定地完成视频真假检测链路，前端也能够将结果以风险标签、置信度和说明文本的形式呈现。这说明，尽管当前尚未在本地部署完整视频伪造检测模型，平台在视频任务上的基础能力已经建立。对毕业设计而言，这样的结果是有价值的，因为它证明了系统并非只支持图像任务，而是已经具备面向多模态内容的统一分析能力。

\subsection{不安全内容审核验证}

不安全内容审核实验同样以链路完整性和结果可解释性为主要考察目标。对图像审核任务而言，平台能够对输入图像返回安全与否、风险标签、建议级别和说明性原因；对视频审核任务而言，平台同样能够生成统一结构的审核结果，并在前端映射为低、中、高不同风险等级。这说明平台在“真假检测”和“安全审核”两类任务之间已经建立了统一的展示逻辑。

结合当前样例验证情况，平台在明显安全场景和明显风险场景下都能够给出符合任务预期的审核提示。从系统层面看，这一实验结果的意义在于：平台已经具备了将多模态审核接口真正转化为可操作功能的能力，而不是停留在单次接口调用层面。也就是说，本文所实现的不安全内容检测模块不仅能“调通”，还能纳入统一风险展示流程之中。

\subsection{功能验证结果讨论}

从视频与安全审核实验可以看出，当前平台已经完成从技术选型、接口接入到前端展示的关键步骤，后续重点在于扩大样本规模和细化定量指标。对于本科毕业设计而言，先完成多模态链路闭环，再逐步扩展本地模型和系统化评测，是与本文系统完成度相匹配的推进方式。

\section{系统性能与可用性分析}

\subsection{响应特征分析}

平台中不同任务的响应模式存在明显差异。图像检测和图像审核任务通常属于相对轻量的同步任务，其结果可以在较短时间内返回，因此更适合直接以同步交互方式呈现。视频生成、图生视频以及部分视频相关理解任务则属于典型的异步长链路任务，需要后端执行轮询和状态封装，再由前端通过加载状态和提示信息等待结果返回。系统当前采用这种“轻任务同步、重任务异步封装”的处理方式，与平台的实际任务特性是匹配的。

从可用性角度看，这种响应组织方式的优势在于，它避免了将复杂的任务状态管理全部压到前端页面中。用户在页面侧只需要感知到“任务正在执行”和“任务已返回结果”这两个主要阶段，而中间的轮询和状态转换由后端统一管理，从而保持了前端交互逻辑的简洁性。

\subsection{平台可用性分析}

平台可用性不仅取决于模型是否能返回结果，还取决于整个交互链路是否清晰。从当前实现来看，平台已经形成了从内容输入、任务选择、结果展示到样本回流的基本闭环，能够覆盖首页导航、内容生成、真假检测、安全审核和结果查看等主要流程。尤其是在前后端职责分离后，不同业务能力虽然来源不同，但在页面层面的交互方式相对统一，这提高了平台整体的一致性。

此外，后端代理层在可用性提升方面发挥了重要作用。它不仅屏蔽了多源接口差异，也通过错误提示和结构化返回降低了用户面对底层异常时的理解成本。可以说，本文平台的可用性并不是单个模型能力直接决定的，而是由“前端交互 + 后端封装 + 模型服务”共同塑造的。

\section{实验结果综合分析}

综合本章实验可以得出以下几点认识。第一，当前平台已经在图像检测、视频检测和不安全内容审核三个方向上形成了完整业务链路，这验证了第三章总体设计和第四章模块实现的合理性。第二，图像任务仍然是当前实验最成熟的部分，本地模型与外部接口的双后端方案为后续开展更严格的量化对比奠定了基础。第三，视频与安全审核任务虽然当前更偏功能验证和案例分析，但其在平台中的接入方式已经稳定，为后续扩展本地视频检测模型和更大规模审核实验提供了入口。

从研究价值看，本文实验并未试图把所有任务一次性做成统一大规模Benchmark，而是结合当前平台阶段，有针对性地验证了最关键的系统能力。这种实验策略的优势在于更贴近当前项目实际，也更有助于支撑毕业设计答辩中的系统展示与方法说明。其不足在于，尚未形成覆盖多种本地视频模型和更大规模风险样本的数据化结论。这一不足将在第六章中进一步总结，并作为后续研究的重要方向。

\section{本章小结}

本章围绕实验设计、阶段性验证结果和系统分析三个层面，对本文平台进行了综合评估。首先，根据当前项目进展，明确了实验取舍原则，将图像虚假检测作为重点定量验证方向，将视频检测与不安全内容审核作为功能验证和案例分析方向。其次，从实验环境、评价指标、数据集与样本构成等方面搭建了统一实验框架，并基于当前平台样本和测试链路完成了图像、视频与安全审核任务的阶段性分析。最后，从响应特征、交互流程和系统组织方式等角度，对平台的可用性进行了总结。

总体而言，实验结果表明本文平台已经完成了从内容生成、内容检测到结果展示的基本闭环，并在多源能力统一组织方面体现出较好的工程可行性。与此同时，实验也揭示了当前系统在视频本地检测、样本规模和大规模定量评估方面仍有提升空间。下一章将在此基础上对全文工作进行总结，并对后续研究方向进行展望。
